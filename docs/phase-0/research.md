## 1. 인스타그램 연동 가능성

### 1.1 현 상황 (학습 데이터 기준)

- **Instagram Basic Display API**: Meta가 2024년 9월 deprecation 공지, **2024년 12월 4일 종료**. 개인 사용자 계정 피드에 외부 앱이 직접 접근하는 마지막 경로가 닫혔다.
- **Instagram Graph API**: 종료 후 남은 공식 경로. 다만 핵심 제약 — **Instagram Business 또는 Creator 계정에 한정**되며, 해당 계정을 Facebook Page에 연결한 상태여야 한다. 일반 사용자(클라이밍 다니는 개인)의 피드는 불가능.
- **oEmbed (Instagram oEmbed API)**: 개별 게시물 단건 임베드. 게시물 URL을 우리 서버에서 oEmbed 엔드포인트로 요청하면 임베드용 HTML 스니펫을 반환. App Review를 통과한 앱에서만 호출 가능.
- **Business Discovery (Graph API)**: 우리(Facebook Page에 연결된 Business Instagram 계정)가 다른 Business 계정의 공개 데이터를 읽을 수 있는 기능. 핵심 제약 — **상대방도 Business/Creator 계정**이어야 함. 일반 Personal 계정은 안 보임.

### 1.2 결론 (요약)

**부분 가능.** 모든 클라이밍장 인스타를 동일하게 받아오는 통일 솔루션은 사실상 **불가능에 가까움**. 그러나 다음 부분 시나리오는 가능:

1. **각 클라이밍장이 Business/Creator 계정으로 전환 + 우리 앱을 Facebook Page에 연결한 경우** → Business Discovery로 피드 최근 게시물 가져오기 가능.
2. **개별 게시물 임베드(oEmbed)** → 가능 (App Review 통과 필요).
3. **클라이밍장 인스타 페이지로 단순 외부 링크 카드** → 100% 가능, 정책 무관.

### 1.3 권장 구현안

- **MVP**: 인스타 통합 시도하지 않음. 각 지점에 `instagram_handle` 컬럼만 두고 **"인스타 보기" 외부 링크 카드**로 처리. 가장 안전, 0 비용, 0 약관 리스크.
- **베타 (V1.1)**: 주요 체인(더클라임, 어센드, 락트리, 클라이밍파크 등)이 Business 계정이라면 우리 앱의 Facebook Page에 연결 요청 → Business Discovery로 최근 9개 게시물 썸네일 그리드 표시. 협업 가능한 체인만 단계적 확장.
- **확장 (V2)**: oEmbed 단건 임베드로 게시판 글 본문 안에 인스타 게시물 임베드 허용. App Review 통과 필요.

### 1.4 대체안

- 각 지점이 우리 사이트에 직접 사진을 업로드 (gym_photos 테이블). 우리가 통제하는 자산이라 가장 안전.
- 사용자 제보 모델 — 사용자가 해당 지점에서 찍은 사진을 게시판에 올리는 흐름과 자연스럽게 연결.
- 추후 인스타 통합이 가능해진 항목만 점진적 추가.

### 1.5 발행 전 라이브 확인 필요
- [ ] Meta Developers — Instagram Graph API 최신 정책 (developers.facebook.com/docs/instagram-api)
- [ ] Business Discovery 사용 가능 권한 범위 변경 여부
- [ ] App Review 최신 절차·소요 기간·필요 자료
- [ ] oEmbed 호출 가격·레이트리미트 현행 (무료 한도)
- [ ] Instagram Platform Terms — 제3자 게시물 표시 약관 조항

## 2. 지도 API 비교 (네이버 vs 카카오 vs 구글)

### 2.1 비교표 (학습 데이터 기준 — 가격·정책은 발행 전 라이브 확인)

| 항목 | 카카오 지도 | 네이버 지도 | 구글 지도 (Google Maps Platform) |
|---|---|---|---|
| 한국 POI 정확도 | **매우 높음** (카카오맵 사용자 풀이 큼) | **매우 높음** (네이버 검색과 연동) | 중간 (해외 사용자 데이터 위주) |
| 마커·클러스터링 | 지원 | 지원 | 지원 (MarkerClusterer 라이브러리) |
| POI 검색 (Local API) | 지원 (키워드·카테고리) | 지원 (지역 검색) | 지원 (Places API) |
| 정적 이미지 (Static Map) | 지원 | 지원 | 지원 |
| 모바일 SDK | iOS/Android | iOS/Android | iOS/Android |
| 무료 한도 | 일일 호출 한도 존재 (학습 데이터 기준 수만 건/일 수준) | 신청·심사 후 무료 한도 | 매월 $200 크레딧 (학습 데이터 기준) |
| 가격 모델 | 호출당 단가, 호출 유형별 차등 | 호출당 단가, 호출 유형별 차등 | 호출당 단가 USD 청구 |
| 상용 사용 가능 | 가능 (사용처·도메인 등록 필요) | 가능 (사용처·도메인 등록 필요) | 가능 |
| 광고 노출 시 제약 | 광고/유료 콘텐츠 위 지도 표시는 카카오 광고와 충돌하지 않는 한 일반적으로 허용 (약관 재확인) | 동일 (약관 재확인) | 허용 |
| 약관 — 데이터 캐싱 | 일정 시간 캐싱 일반적으로 허용 (구체 시간·항목 약관 재확인) | 캐싱 정책 약관 재확인 | Geocoding 결과 최대 30일 캐싱 등 항목별 규정 (학습 데이터 기준) |
| API 키 관리 | 도메인 화이트리스트, 일일 한도 설정 | 동일 | 키 제한 + Referrer/IP 제한 |

### 2.2 권장: **카카오 지도** (차선: 네이버 지도)

근거:
1. **한국 POI 데이터 품질이 핵심**. 실내 클라이밍장 같은 작은 시설은 카카오맵 사용자 제보 기반 POI가 가장 정확.
2. **카카오 로그인을 이미 사용**(소셜 OAuth) → 같은 카카오 개발자 콘솔에서 키 관리, 단일 벤더 관계.
3. **JavaScript SDK가 가장 직관적**, 한국 개발자 커뮤니티의 예제·트러블슈팅 풍부.
4. 비용 — 호출 건수가 폭증하지 않는 한 한국 지도 두 곳(카카오/네이버) 모두 구글보다 저렴할 가능성 큼 (구체 단가는 라이브 확인).

차선이 네이버인 이유: 일부 지방 소도시 POI는 네이버 검색 데이터가 더 풍부할 수 있다. 두 API 모두 비용 모델이 비슷하므로 어느 쪽으로도 큰 차이 없음.

구글은 한국 POI 정확도와 약관 비용 모두 1순위가 되기 어려움. 단, 해외 거주 한국인 사용자 비율이 의미 있을 경우 구글이 fallback으로 가치 있음.

### 2.3 우려사항

- **카카오·네이버 모두 사용처·도메인 등록 + 심사 필요**. 등록한 도메인 외 사용 시 약관 위반.
- **클라이언트 키 노출 불가피** — 도메인 화이트리스트로만 보호. CSP 강화 필요.
- **API 키 일일 한도 초과 시 API 차단** → 마커 표시 실패. 부하 예상치를 일일 한도 미만으로 유지하거나 유료 플랜 가입.
- **Geocoding 결과 캐싱 약관** — 카카오/네이버는 우리 DB에 좌표를 영구 저장하는 것을 어떻게 보는가? `gym_branches.lat/lng` 저장 시 약관 위반인지 확인 필수.

### 2.4 구현 메모

- 지도 라이브러리: 카카오 공식 JavaScript SDK (`dapi.kakao.com/v2/maps/sdk.js`)를 Next.js `<Script>`로 비동기 로드
- 클러스터링: 카카오 ClusterStyle, 사용자 위치 기반 ±10km 마커 그룹
- 정적 이미지: 클라이밍장 상세 페이지 SSR 시 정적 이미지 → 클라이언트 hydration 후 인터랙티브 지도로 교체
- 키: NEXT_PUBLIC_KAKAO_MAP_KEY (클라이언트 노출 불가피, 도메인 제한으로 방어)

### 2.5 발행 전 라이브 확인 필요
- [ ] developers.kakao.com — 지도 API 2026년 5월 가격표·일일 무료 한도
- [ ] developers.kakao.com — 카카오 지도 약관: Geocoding 결과 영구 저장 허용 여부
- [ ] developers.naver.com — 지도 API 가격·약관 동일 확인
- [ ] cloud.google.com/maps-platform — 한국 사용처 + 가격
- [ ] 각 API 키 관리 모범 사례 (도메인 제한, 일일 한도, 알림)

## 3. 클라이밍장 데이터 확보 경로

### 3.1 후보 출처

#### 3.1.1 공공데이터포털 (data.go.kr)
- 문화체육관광부 산하 — 체육시설업 데이터셋 다수 존재 (학습 데이터 기준)
  - "전국 체육시설업체 정보" 등 시군구별 등록 데이터
  - 다만 "실내 클라이밍" 카테고리가 명시적으로 분리되어 있을 가능성은 낮음 — "기타 스포츠시설" 또는 "체육도장업" 등으로 등록되어 필터링 필요
  - **CC-BY 또는 공공누리 4유형 라이선스**일 가능성 높음 — 상용 사용 가능 + 출처 표시 의무 (각 데이터셋 라이선스 페이지 확인 필요)
- 지자체 데이터 — 서울 열린데이터광장(data.seoul.go.kr), 경기도 데이터드림 등에 추가 데이터셋이 있을 수 있음

#### 3.1.2 카카오 / 네이버 장소검색 API로 시드 수집
- "실내 클라이밍" 키워드로 전국 검색 → 결과 좌표·이름·주소를 우리 DB에 시드
- **약관 위험** — 카카오/네이버 모두 "API 결과를 우리 DB로 영구 저장하는 행위"를 명시적으로 금지하거나 제한하는 조항이 있을 가능성. 라이브 확인 필요.
- 합법적 활용: API를 호출해 사용자에게 실시간 표시하는 것은 허용. 그러나 DB 영구 저장은 회색지대.
- 안전한 우회: API 결과를 화면 표시에만 사용하고, **사용자가 "이 지점을 우리 사이트에 추가해주세요" 버튼을 눌러 수동 검증 후 DB 저장** — 사용자 행동에 따른 데이터 입력으로 해석 가능.

#### 3.1.3 한국 주요 실내 클라이밍 체인 (공식 사이트 정보)
- 더클라임 (theclimb.co.kr)
- 어센드 (ascendclimbing.com 또는 ascend.kr 추정)
- 락트리 (rocktree.kr 추정)
- 클라이밍파크 (climbingpark.kr 추정)
- 손상원 클라이밍 짐
- 비레이 클라이밍 짐
- 기타 단일 지점들
- **크롤링 가능성** — robots.txt + 이용약관 확인 필요. 일반적으로 공식 사이트의 지점 정보는 공개 사실이지만 자동 수집은 약관 위반 가능성.
- **법적 위험**:
  - 저작권법: 데이터베이스권 (저작권법 제93조) — 상당한 투자로 만든 DB의 무단 복제 금지
  - 정보통신망법: 자동 접속 + 정상 운영 방해 시 위반
  - 부정경쟁방지법: 영업비밀·성과 무단 사용 금지

#### 3.1.4 수기 입력 + 사용자 제보
- 가장 합법적·안전
- 운영 부담 — 초기 시드 50~100개 지점은 운영진이 수기 입력
- 베타 진입 후 사용자 제보로 확장 — staging 테이블 + 운영자 승인 워크플로우

#### 3.1.5 협회 데이터
- 대한산악·등반스포츠연맹(KASF) 공식 등록 시설 명단이 공개되어 있는지 확인 (학습 데이터 기준 미확인)
- 등록 시설은 신뢰도 높음

### 3.2 단계별 데이터 확보 전략

#### MVP (V1.0) — 시드 50~100개 지점
1. 공공데이터포털 체육시설 데이터셋 다운로드 (CSV·JSON)
2. "실내 클라이밍" / "암벽" 관련 키워드 필터링
3. 운영진이 수기 검증 (전화·홈페이지 확인) 후 `gym_branches` 테이블에 적재
4. 좌표는 카카오 지오코딩 API로 1회 변환 (실시간 사용자 표시용이 아닌 시드 변환이라 약관 회색지대 — 라이브 확인 필요)
5. 사진은 운영진이 공식 인스타·홈페이지에서 출처 명시 후 다운로드 / 또는 운영진이 현장 방문 촬영

#### 베타 (V1.1~) — 확장 500개
1. 사용자 제보 시스템 가동 — `gym_branch_submissions` 테이블 (RLS: 누구나 insert, 관리자만 read+승인)
2. 사용자가 카카오 장소검색을 사이트 내에서 사용 → 결과를 클릭 → 정식 제보 폼 자동 채움 (사용자 행동 기반이라 약관 안전)
3. 관리자 승인 시 `gym_branches`로 복사
4. 사용자에게 포인트 보상 (활동 점수 누적 — `profiles.points`)

#### 정식 (V2~) — 전국 망라 1,000+
1. 협회와 협업 가능성 타진 — 공식 등록 시설 데이터 제공받는 MOU
2. 각 체인 운영사와 협력 — 공식 데이터 직접 제공받기 (대안)
3. 큐레이션 + 사용자 제보 + 협회 데이터의 삼중 검증

### 3.3 법적 위험과 회피

| 위험 | 회피 |
|---|---|
| 크롤링 약관 위반 | 자동 크롤링 안 함. 공공데이터 + 수기 + 사용자 제보 + 협력 데이터만 |
| API 결과 영구 저장 약관 | API는 실시간 표시에만, 시드 좌표는 운영진이 1회 변환 후 DB 영구 저장 가능 여부 약관 재확인 |
| 사진 저작권 | 모든 사진은 출처·라이선스 명시. 본인 촬영 / 공식 제공 / CC-BY만 |
| 데이터베이스권 | 다른 사이트 DB 무단 복제 안 함. 공공데이터 + 자체 수집 |
| 부정경쟁방지법 | 경쟁 사이트의 영업 데이터 무단 활용 금지 |

### 3.4 발행 전 라이브 확인 필요

## 4. 라이선스·법무 종합

### 4.1 상용 사용 시 추가 필요 항목

| 의존성 | 상용 시 필요 |
|---|---|
| Supabase | Pro 플랜 이상 (백업·PITR·SLA) |
| Vercel | Pro 플랜 권장 (custom domain, env 환경 분리, 분석) |
| Sentry | Team 플랜 이상 (보존 기간·시드) |
| 카카오 지도 | 사용처·도메인 등록 (무료) + 일일 한도 초과 시 유료 |
| 네이버 지도 | 등록·심사 (무료) + 유료 한도 |
| Instagram Graph | App Review 통과 (무료) |
| 한국 폰트 (Pretendard) | OFL 라이선스 — 상용 무료 |
| shadcn/ui | MIT — 무료 |
| 모든 npm 패키지 | MIT/Apache/BSD 위주 → 의존성 라이선스 스캔 의무화 (npm audit, license-checker) |

### 4.2 한국 개인정보보호법 (PIPA) 측면

- **지점 정보 자체는 개인정보 아님** (사업장 주소·전화는 영업 정보)
- **운영자(점장) 이름·연락처는 개인정보**일 수 있음 → 사이트에 표시할 때 수집 동의 또는 공개된 정보의 인용으로만
- **사용자 게시글의 좌표·시간 정보 결합 → 개인 식별 가능** → 게시글의 위치 태그는 지점 단위로만, 정확한 좌표 노출 금지
- **수집 항목 최소화** — 이메일·닉네임만으로 운영. 휴대폰 번호·실명은 수집하지 않음
- 영업양도/위탁 처리 — Supabase가 해외 위탁 처리자(미국 본사 / 서울 또는 도쿄 데이터센터)에 해당 → 개인정보처리방침에 명시
- 만 14세 미만 가입 제한 또는 법정대리인 동의 (현행법) → 가입 흐름에 출생연도 입력 + 검증

### 4.3 사용자 제보 콘텐츠의 저작권

- 이용약관에 **"사용자가 게시한 콘텐츠에 대해 운영사는 서비스 운영 목적의 사용권을 가진다"** 조항 필수
- 사용자는 본인 콘텐츠의 저작권 보유 — 운영사는 비독점적·전세계·무상 사용권만
- 사용자 탈퇴 시 게시글 처리 — 익명화 vs 삭제 (둘 다 옵션 제공 권장)
- 제3자(다른 사용자) 사진 무단 게시 → 신고 → DMCA·저작권법 처리 절차 (ops-checklist.md 참조)

### 4.4 약관·개인정보처리방침 초안 필수 항목 (요약, 자세히는 ops-checklist.md)

- 개인정보 수집·이용 항목·목적·보유기간
- 제3자 제공 (없으면 "없음" 명시)
- 위탁 처리자 (Supabase, Vercel, Sentry 등 모두 나열) — 위탁 업무 내용·국외이전 사실
- 정보주체 권리 (열람·정정·삭제·처리정지)
- 개인정보 보호책임자 연락처
